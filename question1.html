Difference between HTTP 1.1 and HTTP 2

HTTP:-

The HTTP protocol can be used to transfer the data in the form of plain text, hypertext, audio, video, and so on.It is a connectionless protocol. HTTP client initiates a request and waits for a response from the server. And also a media independent as data can be sent as long as both the client and server know how to handle the data content. HTTP is a stateless protocol as both the client and server know each other only during the current request.

Differences between HTTP1.1 and HTTP 2:-

As the technology developed and the Internet evolved, HTTP 1.1 soon became outdated. This was the reason why HTTP 2 was developed.

HTTP 1.1 compresses data by itself,while HTTP 2 uses HPACK to compress the data.

HTTP 1.1 uses plain text to encode and transmit the data,while HTTP 2 uses series of binary code.

HTTP 1.1 uses single request and response over single connection,while 2 uses multiple request and response.

1.1 uses cannot handle buffer overflow,whilw 2 handle overflow.

Most modern web browsers support HTTP 1.1 and HTTP 2 including Google Chrome, Mozilla Firefox, Apple Safari, Microsoft Edge, and Opera.
